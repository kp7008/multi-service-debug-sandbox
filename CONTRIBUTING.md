# Contributing to MSDS

Thank you for your interest in contributing to MSDS!

## Contributor License Agreement (CLA)

By contributing to MSDS, you agree that:

1. Your contributions are your original work
2. You grant the project the right to use your code under MIT License
3. Your contributions don't violate anyone else's rights
4. The original creator retains project leadership and direction

## How to Contribute

### Reporting Issues
1. Check if issue already exists
2. Provide clear description and steps to reproduce
3. Include your environment details
4. Add relevant logs or error messages

### Submitting Changes

1. Fork the repository
2. Create a feature branch:
   ```bash
   git checkout -b feature/your-feature-description
   ```
3. Make your changes and test thoroughly
4. Add or update tests for new features
5. Update documentation as needed
6. Commit with clear messages:
   ```bash
   git commit -m "Add feature: clear description of changes"
   ```
7. Push to your fork:
   ```bash
   git push origin feature/your-feature-description
   ```
8. Create a Pull Request with:
   - Clear description of changes
   - Reference to related issues
   - Testing instructions
   - Any breaking changes noted

## Code Style & Standards

- **Python:** Follow PEP 8 guidelines
- **Files:** Use 4 spaces for indentation
- **Comments:** Write clear, concise comments
- **Documentation:** Update docstrings for new functions
- **Testing:** Write tests for new features
- **Linting:** Run `make clean` and `flake8` before committing

## Pull Request Guidelines

- One feature per pull request
- Reference any related issues
- Include tests for new functionality
- Update README if adding features
- Keep commits clean and focused
- Respond to review feedback promptly

## Code of Conduct

We are committed to providing a welcoming and inclusive environment for all contributors.

### Expected Behavior
- Be respectful and professional
- Be open to feedback and ideas
- Listen to other perspectives
- Report issues constructively
- Help others succeed

### Unacceptable Behavior
- Harassment, discrimination, or abuse
- Disrespectful or hostile language
- Unwelcome sexual or violent content
- Spam or self-promotion
- Violation of privacy
