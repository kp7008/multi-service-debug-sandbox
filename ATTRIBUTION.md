# Attribution & Usage Guidelines

## Using MSDS in Your Project

When using MSDS (or derivatives) in your project, please follow these guidelines:

### 1. Include Copyright Notice

Add to your project's README:

```
This project uses MSDS - Multi-Service Debug Sandbox
Copyright (c) 2026 Pratik Koshiya
Licensed under MIT License
https://github.com/kp7008/msds
```

### 2. Include License File

- Distribute the MIT License file
- Maintain original copyright notice
- Keep copyright notice visible

### 3. Documentation

- Link to MSDS project in your documentation
- Credit the original creator
- Mention where you used/modified MSDS patterns

### 4. For Commercial Use

If using MSDS commercially:

✅ Include LICENSE file  
✅ Keep copyright notice  
✅ Link back to original project  
✅ No additional permission needed (MIT License)

## What NOT To Do

❌ Don't claim you created MSDS  
❌ Don't remove copyright notices

❌ Don't violate the MIT License terms  

## Questions About Attribution?

If you're unsure about proper attribution:

1. Include the LICENSE file - that covers you
2. Add link to MSDS project
3. Mention Pratik Koshiya as creator
4. Keep it visible and findable

When in doubt, more attribution is better than less!
